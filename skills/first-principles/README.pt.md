# First Principles Analysis

## O que faz

Decompõe problemas complexos ou ambíguos em seus fundamentos antes de implementar. Força clareza de pensamento através de um processo estruturado de 6 etapas: definição do problema, identificação de restrições, decomposição em subproblemas, exploração de abordagens, avaliação de trade-offs e decisão fundamentada. Ideal para evitar implementações precipitadas em contextos de alta incerteza.

## Como invocar

```
/first-principles <descrição do problema>
```

**Exemplos:**
- `/first-principles redesenhar autenticação do sistema legado`
- `/first-principles otimizar pipeline de dados que processa 10GB diários`
- `/first-principles arquitetura de cache distribuído para microserviços`

## Quando usar

- Problemas com requisitos ambíguos ou mal definidos
- Decisões arquiteturais com impacto de longo prazo
- Quando há múltiplas abordagens viáveis e incerteza sobre qual escolher
- Antes de refatorações ou redesenhos significativos em código existente
