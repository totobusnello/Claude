# ADR-006: D41 — 5 cross-cutting decisions for Q/A/P implementation

Date: 2026-05-18

## Status

Accepted

## Context

After the overnight multi-agent push of 2026-05-17 (15 PRs opened covering all Q/A/P pillars), five cross-cutting implementation questions were identified that could not be resolved within individual PRs. These questions blocked implementation decisions across multiple branches and needed a single authoritative answer before merged work could proceed coherently:

1. **Which Gemini model is the default for P1 (answer primitive)?** Flash vs flash-lite involves a quality/cost tradeoff. The implementation spec (PR #3) left it open.

2. **Should A2 export encrypt by default or require explicit opt-in?** The Autonomy pillar's positioning ("your data, your choice") has implications for what "default secure" means.

3. **Which visual brand palette for GTM assets?** Four candidates (A: expressive, B: vibrant, C: data-viz, D: minimal mono+accent) were generated by the asset production agent. The choice affects banner, SVGs, logo, and all future marketing collateral.

4. **What is the minimum absolute lift threshold for L3 confidence field activation?** Too low risks complexity for marginal gain; too high risks never activating.

5. **What is the implementation sprint order?** P1→A2→P2→P4 or some other sequence. Order matters because P2 (hooks auto-capture) depends on P1's answer primitive being stable, and A2 (export) is backend-heavy enough to run in parallel.

These 5 questions were resolved by Toto during the morning review of 2026-05-18 (~06:00 BRT) in response to `docs/MORNING-REVIEW-2026-05-18.md`.

## Decision

**1. P1 default Gemini model: `gemini-2.5-flash-lite`**
Cost is the primary criterion ("tem que ser barato e bom"). Toggle available post-Q1 to `gemini-2.5-flash` via `--model` flag if an empirical quality issue emerges. PR #3 spec §9 updated to reflect lite as default.

**2. A2 encryption: opt-out (encrypted by default)**
A2 is the keystone Autonomy pillar feature. Encrypted-by-default signals "your data is protected by default" — consistent with the data autonomy moat (ADR-005). Implementation: AES-256-GCM + scrypt KDF. Plaintext available via explicit `--unencrypted` flag for edge cases (trusted local media backup). PR #9 spec §3 reframes default behavior accordingly.

**3. GTM brand palette: D (minimal — mono + 1 accent)**
Rationale: moat is "lean, your data, your choice" — visual minimalism matches the positioning, does not compete with content. Accent color: `#00C896` (success green). Works on dark + light backgrounds. Semantic fit: "data is healthy/safe." Unclaimed in the space (memanto and agentmemory both use orange/amber). Asset production (banner + 6 stat SVGs + logo) completed in PR #19.

**4. L3 confidence gate threshold: ≥1.0pp absolute lift (kept)**
1.0pp is an honest, non-trivial bar. Below that, the ranking complexity is not justified by the margin. If eval returns <1.0pp, schema ships alone (v19) and ranking integration waits for L3.2 (separate iteration). PR #15 spec §6 unchanged.

**5. Implementation sprint order: P1 → A2 (parallel if capacity) → P2 → P4**
P1 = highest user impact, unblocks P2-P5 mental model. A2 = backend-heavy, safe to run in parallel with P1 if capacity allows. P2 depends on P1 stable API. P4 depends on P2 hooks. Implementation kickoff issues created: P1 (PR #18), A2 (PR #17), P2 (in progress), A3 (in progress), P4 (in progress).

## Consequences

- **Positive:** All 5 questions resolved before implementation merges; no cross-branch inconsistency. PRs #3, #9, #15, #16 updated with PR comments encoding the decisions.
- **Positive:** Palette D + accent `#00C896` locks visual identity across all GTM assets before any external publication.
- **Negative:** `gemini-2.5-flash-lite` as P1 default may produce lower-quality answers than flash; if quality issues surface post-Q1, a flag-change deploy is needed.
- **Negative:** Encrypted-by-default in A2 increases onboarding friction slightly (user must manage a passphrase for exports); mitigated by `--unencrypted` escape hatch.
- **Risks:** Sprint order assumes P1 can be implemented stably before P2 begins; if P1 API changes during P2 work, P2 integration needs rework.

## Alternatives considered

**Decision 1 (model):**
- `gemini-2.5-flash` as default — rejected: quota risk (3M/d threshold known from 2026-04-20 incident); cost-conscious operation is the guiding principle for agent infra.

**Decision 2 (encryption):**
- Opt-in (plaintext by default) — rejected: contradicts autonomy positioning; a user who exports without realizing their data is plaintext undermines the "your data, protected" narrative.

**Decision 3 (palette):**
- Palette A (expressive) — rejected: too loud for "lean" positioning.
- Palette B (vibrant) — rejected: too similar to competitor color schemes.
- Palette C (data-viz) — rejected: too technical/corporate for the indie dev audience.

**Decision 4 (threshold):**
- 0.5pp — rejected: too easy to hit with noise; leads to premature activation.
- 2.0pp — rejected: too conservative; L3 would likely never activate.

**Decision 5 (sprint order):**
- A2 first (export is "foundational") — rejected: P1 (answer primitive) is higher perceived user value and unblocks more downstream work.
- All in parallel — rejected: P2 and P4 have hard dependencies on P1; parallel execution would cause rework.
