# benchmarks/latency-cost — Production SOTA Benchmark

Measures nox-mem production characteristics (latency, cost, RAM, throughput)
and compares against published Zep / Mem0 / MemOS / LangMem / Letta numbers.

## Files

| File | Purpose |
|------|---------|
| `latency.py` | HTTP API latency benchmark — p50/p95/p99 across 4 query configs |
| `cost.py` | Per-query cost calculator + competitor pricing comparison |
| `footprint.py` | RAM/throughput measurement (live or cached) |
| `RESULTS-PRODUCTION-SOTA.md` | Human-readable comparative report |
| `results/RESULTS-PRODUCTION-SOTA.json` | Machine-readable results + methodology |
| `results/latency_raw.json` | Raw latency run output (generated by latency.py) |
| `results/cost_analysis.json` | Cost analysis output (generated by cost.py) |
| `results/footprint.json` | Footprint output (generated by footprint.py) |

## Quick Start

### From VPS (live measurements)

```bash
# Source env first (mandatory)
set -a; source /root/.openclaw/.env; set +a

# Latency benchmark (requires nox-mem API running on :18802)
python benchmarks/latency-cost/latency.py \
  --api-url http://127.0.0.1:18802 \
  --n 100 \
  --output benchmarks/latency-cost/results/latency_raw.json

# Cost analysis (static calculation, no API required)
python benchmarks/latency-cost/cost.py \
  --output benchmarks/latency-cost/results/cost_analysis.json

# Footprint (requires /proc access for RSS measurement)
python benchmarks/latency-cost/footprint.py \
  --api-url http://127.0.0.1:18802 \
  --output benchmarks/latency-cost/results/footprint.json
```

### From local machine (cached measurements)

```bash
# Uses pre-measured values from 2026-05-29 VPS run
python benchmarks/latency-cost/footprint.py --cached

# Cost analysis runs anywhere
python benchmarks/latency-cost/cost.py
```

## Methodology

- All measurements via HTTP API (`/api/search`, `/api/kg/path`)
- 10-iteration warmup discarded
- Nearest-rank percentiles (no trimming)
- Production 69,135-chunk corpus (real VPS data, not synthetic)
- VPS: Hostinger BR, 4vCPU, 16GB RAM, Node.js v22.22.2

## Latency Caveat

The p50=529ms is **dominated by the Gemini embed API round-trip**
(~400-600ms from Hostinger Brazil to Google US-East). Without embed:
- FTS5 BM25: 2-5ms
- KG walk: 2-5ms
- RRF merge: <1ms
- **Total without embed: ~10-15ms**

With a local embed model, nox-mem would reach ~15-50ms p50.
