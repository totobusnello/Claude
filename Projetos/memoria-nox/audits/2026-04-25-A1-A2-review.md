## Fixes implementados (detalhamento técnico)

### Fix #1 — Filename collision-resistant
**Antes:** `${opName}-${ts.substring(0,14)}.db` (granularidade 1 segundo)
**Depois:** `${opName}-${ts}-${process.pid}-${randomUUID().slice(0,8)}.db`

### Fix #2 — Fail-closed semantics
**Antes:** `snapshot()` retorna `null` em falha, op prossegue.
**Depois:** `snapshot()` throws em qualquer falha. `withOpAudit` propaga error a menos que `NOX_ALLOW_NO_SNAPSHOT=1` esteja explícito.

### Fix #3 — Path traversal protection
- `getValidatedSnapshotDir()` resolve path absoluto, valida prefix em allowlist (`/var/backups/`, `/root/.openclaw/`).
- `opName` validado contra regex `^[a-z0-9-]{1,32}$` antes de qualquer uso.

### Fix #4 — Zombie reaper
- `reapZombies()` exportado.
- Marca `status='crashed', error_message='reaped: process died before completing'` em rows com `status='running' AND started_at < now-1h`.
- Caller responsável por chamar no startup (TODO doc no README).

### Fix #5 — VACUUM INTO atomicity real
**Antes:** `db.prepare("VACUUM INTO ?").run(finalPath)` — escrita direta no destino, kill mid-write deixa arquivo parcial.
**Depois:** sequência atômica:
```
1. VACUUM INTO `${path}.tmp`
2. Open .tmp readonly + PRAGMA integrity_check
3. Se ok: renameSync(.tmp → finalPath) (atomic em mesmo FS)
4. Se fail: unlink .tmp + throw
```

### Fix #6 — `safeRestore()` helper exportado
- Valida snapshot integrity_check antes de restaurar
- Valida `user_version` match (throws se diferente, override via `{ force: true }`)
- Remove `<DB>-wal` e `<DB>-shm` órfãos
- Restore via `VACUUM INTO tmpDest` + `renameSync`
- Retorna `{ ok: true, warnings: [...] }`

### Fix #7 — Schema version tracked
- `ops_audit` ganhou colunas `schema_user_version`, `pid`, `snapshot_bytes`
- Cada audit row registra `PRAGMA user_version` no momento da op

### Fix #8 — Guard duplicado removido
- `src/ingest.ts` tinha guard que duplicava lógica do router
- Removido. Comentário aponta pra router canônico.
- Defesa em camadas mantida: callers DEVEM usar `routeIngest()` (4 callers já refatorados em A2).
- Edge case fallback: `routeIngest` em entity-shaped com parse fail chama `ingestFile` direto (não passa pelo guard que não existe mais).
