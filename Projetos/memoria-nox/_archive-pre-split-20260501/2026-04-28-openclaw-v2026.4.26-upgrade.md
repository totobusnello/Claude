## Phase 8 — Wins Opt-in v.26 (separado — só após 7d estável)

> Princípio v.25 mantido: simplificação só após estabilidade.

### 8.1 Compaction preflight (`agents.defaults.compaction.maxActiveTranscriptBytes`)

**O que faz:** quando JSONL ativo passa o byte limit, força local compaction antes do próximo turn — successor file menor, evita blowup de transcripts.

**Pré-condição:** medir tamanho atual dos transcripts dos 6 agents.
```bash
ssh root@100.87.8.44 'for a in nox forge atlas boris cipher lex; do
  echo -n "$a: "
  ls -la /root/.openclaw/agents/$a/sessions/*.jsonl 2>/dev/null | awk "{print \$5}" | sort -rn | head -3
done'
```

**Quando ativar:** se algum agent tem transcript >50MB ativo.

```bash
# Exemplo de threshold: 30MB
ssh root@100.87.8.44 'openclaw config set agents.defaults.compaction.maxActiveTranscriptBytes 31457280 && \
  openclaw config validate'
ssh root@100.87.8.44 'systemctl reload openclaw-gateway || systemctl restart openclaw-gateway'
```

### 8.2 Asymmetric memory embeddings (relevante pro nox-mem)

**Mudança v.26:** `memorySearch.inputType`, `queryInputType`, `documentInputType` permitem diferenciar embeddings de query vs documento — Gemini suporta nativamente (`task_type=RETRIEVAL_QUERY` vs `RETRIEVAL_DOCUMENT`).

**Hoje no nox-mem:** chamamos Gemini com `task_type=RETRIEVAL_DOCUMENT` em ambos. Ranking poderia melhorar se queries usassem `RETRIEVAL_QUERY`.

**Avaliação:** rodar shadow-mode 7d com asymmetric e comparar precisão de retrieval contra baseline simétrico atual. Não ativar diretamente em prod.

### 8.3 Re-validar `OPENCLAW_NO_RESPAWN`

Mesma da v.25 Phase 8.4 — se #62028 não voltou em 30 dias pós-v.25 + v.26, considerar simplificar wrapper.

### 8.4 NÃO fazer: Cerebras

Não temos use case. Mantém disabled.

### 8.5 NÃO fazer: migrate-claude / migrate-hermes

Importadores pra config externa. Não usamos.
